    rubyを導入しようと思いRubyの基本について学習。
    開発環境にRubyをインストールしようと思ったが、Rubyが無ければならない。 他の環境で、毎回Rubyの環境を用意しなければならない。
    Javaであれば、どこでも動かせるメリットがある。
    これを機に、Javaの基本を学び実際に導入してみたい。
    RubyではMVCモデルで作成したが…
    M:クラス。javaクラスのこと。様々な処理を行うプログラム。
    V:jsp。java server pages。表示部分。
    C:サーブレット。司令塔。

    JavaでのWeb制作に必要なもの。
    Visual Studio Code : エディタ。jsp役。
    eclipse : Web開発用統合開発環境。
    MySQL Workbench : データベースを簡易に操作。
    Apache Tomcat : サーブレットコンテナ。Webサーバ。必須。
    流れ
    1. HTML, CSSを作成
    2. 1をもとにjspを作成
    3. jspに出力またはjspから入力されるデータに基づくDBを作成
    4. クラスとサーブレットでjspの入出力を管理する。
    文字列型Stringと文字型Charは違うので注意。
    "String", 'Char'これも違うルール。
    なお、ここのページにbootstrapが適用されていないので、grid適用されていない。

    別の導入方法流れ

    https://note.com/liber_grp/n/n88f3f0a6fdf1
    1. JDK（インストーラ）をインストール、実行。
    2. 「Extension Pack For Java」という拡張機能をインストール 3.JavaHomeを設定する
    JavaHOMEとは、初めにインストールしたJDKの場所をVSCodeへ知らせてあげることを指します。
    これにより、書いたプログラムをコンパイルしたり、コンパイルして出てきた実行ファイルを実行することが出来る様になります
